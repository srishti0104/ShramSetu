# Storage Configuration Guide

## S3 Buckets

### 1. Audio Bucket (`shram-setu-audio`)

**Purpose**: Store voice recordings for grievances and voice commands

**Configuration**:
- Encryption: AWS KMS (customer-managed key)
- Versioning: Enabled
- Public Access: Blocked
- Lifecycle Policy:
  - Move to Intelligent Tiering after 30 days
  - Move to Glacier after 180 days
  - Delete after 365 days

**Folder Structure**:
```
shram-setu-audio/
├── grievances/
│   └── {userId}/
│       └── {grievanceId}/
│           └── audio.mp3
├── voice-commands/
│   └── {userId}/
│       └── {sessionId}/
│           └── command.mp3
└── transcriptions/
    └── {userId}/
        └── {audioId}/
            └── transcript.json
```

**Allowed File Types**: .mp3, .wav, .m4a, .ogg, .webm
**Max File Size**: 10MB

### 2. Document Bucket (`shram-setu-documents`)

**Purpose**: Store payslip images, receipts, and verification documents

**Configuration**:
- Encryption: AWS KMS (customer-managed key)
- Versioning: Enabled
- Public Access: Blocked
- Lifecycle Policy:
  - Move to Intelligent Tiering after 90 days
  - No expiration (keep for compliance)

**Folder Structure**:
```
shram-setu-documents/
├── payslips/
│   └── {workerId}/
│       └── {year}/
│           └── {month}/
│               └── payslip.jpg
├── receipts/
│   └── {transactionId}/
│       └── receipt.jpg
├── verification/
│   └── {userId}/
│       └── document.pdf
└── eshram/
    └── {userId}/
        └── eshram-card.jpg
```

**Allowed File Types**: .jpg, .jpeg, .png, .webp, .pdf
**Max File Size**: 5MB

**Upload URLs**:
- Expiration: 5 minutes
- Max file size: 5MB
- Generated by Lambda functions

**Download URLs**:
- Expiration: 1 hour
- Generated on-demand

## ElastiCache Redis

### Cluster Configuration

**Instance**: cache.t3.micro
**Engine**: Redis 7.0
**Port**: 6379
**Encryption**:
- In transit: TLS enabled
- At rest: Enabled
- AUTH token: Required

### Data Structures

#### 1. Session Management
```
Key: session:{userId}
Type: Hash
TTL: 1 hour
Fields: userId, token, lastActivity, deviceInfo
```

#### 2. TOTP Codes
```
Key: totp:{sessionId}
Type: String
TTL: 5 minutes
Value: 6-digit code
```

#### 3. Rate Limiting
```
Key: ratelimit:{userId}:{action}
Type: String
TTL: 1 minute
Value: request count
```

#### 4. OTP Verification
```
Key: otp:{mobileNumber}
Type: Hash
TTL: 5 minutes
Fields: code, attempts, createdAt
```

#### 5. Voice Session Context
```
Key: voice:{sessionId}
Type: Hash
TTL: 30 minutes
Fields: userId, language, currentIntent, history
```

#### 6. Sync Queue Status
```
Key: sync:{userId}
Type: Hash
TTL: 1 hour
Fields: lastSyncTime, pendingOps, failedOps
```

#### 7. Active Work Sessions
```
Key: worksession:{sessionId}
Type: Hash
TTL: 8 hours
Fields: contractorId, jobId, startTime, expectedWorkers
```

### Redis Operations

**Session Management**:
```javascript
// Create session
await redis.hset(`session:${userId}`, {
  userId,
  token,
  lastActivity: Date.now(),
  deviceInfo: JSON.stringify(device)
});
await redis.expire(`session:${userId}`, 3600);

// Get session
const session = await redis.hgetall(`session:${userId}`);

// Delete session
await redis.del(`session:${userId}`);
```

**TOTP Management**:
```javascript
// Store TOTP
await redis.setex(`totp:${sessionId}`, 300, code);

// Get TOTP
const code = await redis.get(`totp:${sessionId}`);

// Delete TOTP
await redis.del(`totp:${sessionId}`);
```

**Rate Limiting**:
```javascript
// Increment counter
const count = await redis.incr(`ratelimit:${userId}:${action}`);
if (count === 1) {
  await redis.expire(`ratelimit:${userId}:${action}`, 60);
}

// Check limit
if (count > MAX_REQUESTS) {
  throw new Error('Rate limit exceeded');
}
```

### Connection Configuration

**Lambda Functions**:
```javascript
import { createClient } from 'redis';

const redis = createClient({
  socket: {
    host: process.env.REDIS_ENDPOINT,
    port: 6379,
    tls: true,
  },
  password: process.env.REDIS_AUTH_TOKEN,
  commandsQueueMaxLength: 1000,
});

await redis.connect();
```

**Alarms**:
- High CPU (>75% for 10 minutes)
- High Memory (>90% for 10 minutes)
- High Evictions (>1000 per 5 minutes)

### Best Practices

**Key Design**:
- Use consistent naming: `entity:id:attribute`
- Always set TTL to prevent memory bloat
- Use descriptive key names

**Performance**:
- Use connection pooling
- Pipeline multiple commands
- Avoid KEYS command (use SCAN)
- Set reasonable timeouts

**Security**:
- Enable AUTH token
- Use TLS encryption
- Rotate tokens regularly
- Restrict VPC access

## Backup and Recovery

### S3
- Versioning enabled for all buckets
- Cross-region replication (optional)
- Lifecycle policies for cost optimization

### Redis
- Daily snapshots (5-day retention)
- Snapshot window: 03:00-05:00 UTC
- Automatic failover (if Multi-AZ enabled)

## Cost Optimization

### S3
- Intelligent Tiering for automatic cost optimization
- Glacier for long-term archival
- Lifecycle policies to delete old data

### Redis
- t3.micro instance for development
- Scale up to t3.small/medium for production
- Monitor evictions to right-size memory

## Security

### S3
- All buckets encrypted with KMS
- Block all public access
- Pre-signed URLs for temporary access
- CloudTrail logging enabled

### Redis
- VPC isolation
- TLS encryption in transit
- Encryption at rest
- AUTH token required
- Security group restrictions
